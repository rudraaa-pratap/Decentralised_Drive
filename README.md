# Decentralized Drive (D-Drive) 🚀

A Web3 file storage application that allows users to securely upload files to IPFS and selectively share access with other Ethereum addresses. Built with React, Solidity, Ethers.js, and Pinata.

**Live Demo:** [https://decentralised-drive.vercel.app/](https://decentralised-drive.vercel.app/)

![Decentralized Drive Preview](https://bafybeihxvq6g3rhlcvj4eqeqy5h3tmdw3i2k27r655x42h5zmsq2h4g56e.ipfs.dweb.link/)

## ✨ Features

- **Decentralized Storage:** Files are securely uploaded to the InterPlanetary File System (IPFS) via Pinata.
- **Smart Contract Access Control:** Only the owner of a file and specifically authorized Ethereum addresses can view the file URLs.
- **Share & Revoke:** Cryptographically grant access to other users by their wallet address.
- **Web3 Authentication:** Instant, passwordless login using MetaMask.

## 📖 How to Use

1. **Connect Wallet:** Visit the live site and click "Connect Wallet". Approve the connection in MetaMask (ensure you are on the Polygon Amoy testnet or Localhost depending on deployment).
2. **Upload a File:** Click "Choose File", select an image or document, and click "Upload to IPFS". Sign the MetaMask transaction to save the IPFS hash to the blockchain.
3. **Share Access:** To allow someone else to view your files, copy their Ethereum wallet address, paste it into the "Share Access" box, and click "Share". Confirm the transaction.
4. **View Shared Drive:** If someone has shared their files with you, enter *their* Ethereum address into the "View Shared Drive" box and click "View Files" to load their IPFS images.

- **Smart Contracts:** Solidity

## 🚀 Running Locally

To run this project on your own machine, follow these steps:

### 1. Clone the repository
```bash
git clone https://github.com/rudraaa-pratap/Decentralised_Drive.git
cd Decentralised_Drive
```

### 2. Install Dependencies
Install dependencies for both the Hardhat backend and the React frontend.
```bash
# Install Hardhat dependencies
npm install

# Install React dependencies
cd client
npm install
cd ..
```

### 3. Start Local Blockchain
Start a local Hardhat node. This will generate 20 test accounts with 10,000 test ETH each.
```bash
npx hardhat node
```

### 4. Deploy Smart Contract
Open a *new* terminal window and deploy the smart contract to your local network.
```bash
npx hardhat run scripts/deploy.js --network localhost
```
*Note the deployed contract address. It should match the `CONTRACT_ADDRESS` in `client/src/App.js`.*

### 5. Setup Pinata (IPFS)
1. Create a free account at [Pinata](https://pinata.cloud/).
2. Generate an API Key and Secret.
3. Open `client/src/App.js` and paste your keys into the `PINATA_API_KEY` and `PINATA_SECRET_KEY` variables.

### 6. Start the Frontend
Start the React development server.
```bash
cd client
npm start
```
The application will be running at `http://localhost:3000`.

### 7. Connect MetaMask
- Open the MetaMask extension and switch to the **Localhost 8545** (or Hardhat) network.
- Import one of the private keys generated by your Hardhat node in Step 3.
- Connect your wallet to the app and start uploading!

## 🔐 Smart Contract (Upload.sol)

The smart contract stores a mapping of user addresses to an array of IPFS image hashes. It also maintains an access list, allowing users to grant viewing permissions to other addresses.

```solidity
function add(address _user, string memory url) external;
function allow(address user) external;
function disallow(address user) external;
function display(address _user) external view returns(string[] memory);
```
